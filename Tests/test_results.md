|:---|:---|:---|
| **TC-AI-01** | При первой попытке часто возвращается ошибка "The model is overloaded. Please try again later." (Модель перегружена). Задача создается только со второй или третьей попытки. | **fail** |

## Сводка по дефектам

| ID Дефекта | Приоритет | Описание |
|:---|:---|:---|
| **BUG-AI-01** | **Высокий** | **Нестабильность API Gemini.** Запросы к ИИ-ассистенту часто завершаются ошибкой "The model is overloaded", требуя от пользователя нескольких попыток для получения результата. |
| **BUG-AI-02** | **Средний** | **Неудобный интерфейс переноса задач через ИИ.** Для переноса задачи ассистент требует от пользователя указать ID задачи, который не отображается в интерфейсе, что делает функцию практически неиспользуемой. |
---

## Выводы (Conclusion)

Тестирование показало, что основной функционал приложения (аутентификация, CRUD задач, расчет шкалы выгорания) работает стабильно и в соответствии с требованиями.

Однако модуль **ИИ-Ассистента** имеет серьезные проблемы, связанные как с нестабильностью внешнего API, так и с недостатками в проектировании взаимодействия.

**Рекомендации:**
1.  **Для BUG-AI-01:** Реализовать на стороне серверной функции `ask-gemini` механизм автоматических повторных попыток (retry) при получении ошибки о перегрузке.
2.  **Для BUG-AI-02:** Улучшить промпт и логику `ask-gemini`, чтобы ИИ мог сам находить ID задачи в базе данных по ее названию, которое предоставляет пользователь.
3.  Провести регрессионное тестирование модуля ИИ после внесения исправлений.
